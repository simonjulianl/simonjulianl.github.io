Hi guys, as mastery check and midterm is coming soon, please use this list of questions to check your understanding. Reflect on what you understand (and how deep do you understand the topic is) and what you don’t understand!

Anyway, there are 3 categories for the questions: fundamental, intermediate and advanced.

  • Fundamental are topics and questions that you must be able to answer.
  • Intermediate are questions that you must be able to answer if you want to obtain at least A grade.
  • Advanced are really just questions to probe your understanding and would give the A+ grade

Reading assessment is really very difficult. The midterm and final exam won’t be that difficult that’s why some of the RA questions appears in the Intermediate and Advanced categories, not in fundamental. Fundamental is really just understanding the concepts at the very basic level.

You can download it here as a word document instead, and it’s formatted as a table already!

Fundamental#

Scope#

  1. What are the different types of declarations in Source?
  2. What are the differences between name declarations and name occurrences, function declarations with function applications?
  3. In sub model, how do you replace the name occurrences (aka how do you determine what each occurrence refers to)

Higher Order Functions#

  1. How do you differentiate which function are higher-order functions?
  2. How can you transform a function declaration into a constant declaration? And is it possible for all functions to be declared as constant declaration?
  3. What are the differences between these two expressions, g(f) and g(f(2))?

Substitution Model & Processes#

  1. Explain the differences between iterative and recursive processes! What are their implications programmes such as space and time complexity, code length, code quality, etc
  2. Explain the differences between applicative and normal order reductions then explains what the implications on the evaluation of the programme are!
  3. Consider this, Does this perform any addition ?

Intermediate#

Scope#

  1. Match these naming occurrences (From RA 21)
  2. What are the differences between function declaration and const declaration? Hint: nested function (maybe there are others too)
  3. Which scope does math_e, constant belong to? And is it possible to redefine math_e in Source Language by using some of the languages construct (such as {}, const, recursion, etc)? If it is, how?

Higher Order Functions#

  1. How do you do recursion, using only lambda construct without any function declaration? To demonstrate your understanding, try to write a factorial function using const declaration only!

Substitution Model & Processes#

  1. Can switching the order of operands reduce the number of deferred operations? (Source evaluates from left to right for operations that have equal precedence)
  2. If a program gives rise to a recursive process, can you transform it into a program that gives rise to an iterative process by switching the operands of additions / subtractions / multiplications?

Advanced#

Scope#

  1. In your substitution model, you always replace the function application with the function body (or the code within {}). If you think about it further, very rarely if not none there is any programming language that uses sub model as the mental evaluation model, why? Are there any gaps in sub model? Or can you justify that the model is perfect

  2. Does this work? State your reasoning! What if I moved the fun(x + y) above const z = 10; Will this work?

  3. Modified from RA 21, will this work? What kind of error would it give if it doesn’t work or prove me that it works!

  1. Consider this lambda function What is the result of the evaluation of this programme?

  2. HOF can be used to implement data structures such as pair, consider

    const p = (x, y) => (flag) => flag < 1 ? x : y;

    This is a very simple implementation of pair purely using flag (and not using other constructs). With this implementation of pair, how then will you implement the head, tail and a list using this pair (aka list (…))

Substitution Model & Processes#

  2. Taken from RA 21, Does this process give rise to iterative or recursive process? And if it’s a recursive process, what are the deferred operations here?
